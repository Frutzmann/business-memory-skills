# Skill : Reporting mensuel

## Quand utiliser ce skill

Quand on te demande de préparer un rapport, faire un point d'activité, ou générer un reporting pour un client ou en interne.

## Instruction

1. Identifie le type de rapport demandé (interne ou client)
2. Récupère les données du client dans la base de données si c'est un rapport client
3. Applique le format et les préférences du client (langue, niveau de détail, format de livraison)
4. Respecte strictement la structure définie ci-dessous

## Rapport interne — "Pulse Mensuel"

Structure obligatoire, dans cet ordre :
1. **Résumé exécutif** — 5 lignes max. CA vs objectif, jours facturés, événement marquant, prévision M+1, alerte si applicable.
2. **Performance financière** — CA facturé, CA encaissé, taux d'utilisation (cible >75%), DSO (cible <35j), marge brute (cible >65%).
3. **Suivi clients** — Par client : jours consommés/budgétés, statut (🟢🟡🔴), prochaine échéance.
4. **Pipeline commercial** — Prospects avec probabilité pondérée. Un prospect = minimum un premier RDV effectué.
5. **Trésorerie** — Solde, prévisions encaissements/décaissements M+1, alerte si solde prévisionnel < 15 000 €.
6. **Actions M+1** — 3 à 5 actions prioritaires.

## Rapport client

Consulter la fiche client pour :
- La fréquence (mensuel, trimestriel, sur demande)
- Le format (email court, PDF brandé, présentation)
- La langue
- Les destinataires
- Les préférences spécifiques (ex: certains clients veulent du ROI chiffré, d'autres veulent du narratif)

## Règles

- Chiffres : toujours avec séparateur de milliers. Pourcentages à une décimale.
- Graphiques : barres ou lignes uniquement. Jamais de camemberts. Jamais de 3D.
- Couleurs : #9D4F9E (violet), #011638 (bleu marine), #EEF0F2 (gris clair).
- Ne jamais envoyer un rapport sans relecture. Si incertitude sur un chiffre, le signaler.
